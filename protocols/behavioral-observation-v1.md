# Behavioral Observation Protocol v1

## Purpose

Observe an AI-native design tool without inferring inaccessible internal mechanisms.

## Preparation

1. Record date, product, visible version, account context, and enabled features.
2. Start from a documented initial state.
3. Define the prompt and variables before the run.
4. Prepare screen capture and artifact export where permitted.

## Procedure

1. Submit the pre-registered input verbatim.
2. Record each agent question, human response, visible state change, and generated artifact in order.
3. Do not add unplanned guidance unless the experiment tests intervention; log every deviation.
4. Export or capture outputs at meaningful checkpoints.
5. Hash or otherwise identify evidence files when practical.

## Analysis rule

Transcribe observable behavior as facts first. Add interpretations only after the factual record is complete. A claim about hidden reasoning must be labeled as a hypothesis and paired with a possible falsification test.

## Minimum reproducibility record

- Exact inputs and human responses
- Initial state and environment
- Sequence and timestamps
- Output artifacts
- Deviations and failures
- Evidence manifest
