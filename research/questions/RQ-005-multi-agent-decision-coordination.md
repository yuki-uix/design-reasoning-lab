# RQ-005: How do concurrent agents divide design decisions?

## Question

When multiple agents generate on the same canvas concurrently, how is the brief partitioned between them, which decisions are shared versus local to one agent, and what mechanisms does the human have to own or revise the merged result?

## Motivation

[RQ-003](RQ-003-decision-ownership.md) modeled ownership as a two-party negotiation between one human and one agent ([FND-004](../../findings/FND-004-ownership-mechanism-stack.md)). Pencil.dev ships concurrent agent teams (vendor claims up to six agents with per-agent canvas indicators), which introduces an agent-to-agent decision boundary the lab has not observed anywhere: cross-cutting decisions like palette and type must either be coordinated, duplicated, or allowed to drift between sections.

## Observable indicators

- How a single-page brief is split (by section, by layer, by role) and whether the split is visible before generation
- Consistency of cross-cutting decisions (palette, typography, spacing scale) across sections generated by different agents
- Visible coordination artifacts: shared plans, locks, sequencing, per-agent status indicators
- Behavior on conflict: repair, overwrite, or persistent inconsistency
- Whether human feedback can target one agent's output specifically, and who fields it
